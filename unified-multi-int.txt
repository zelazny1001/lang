Create an action diagram (or flowchart) that can be displayed in PlantUML to capture the following logic:
When a transition from customer to agent occurs:
1. String transitionToReset = assemble the utterances from transition until the previous reset

The input data for the assembly step above will be a sessionID and a list of AAResponse objects

2. String callPurposeForReset = invoke call purpose summarization on the String transitionToReset

3. String callPurposeForResetIntent = invoke intent prediction on the callPurposeForReset String

4. If callPurposeForResetIntent is present in the intent white list
boolean sessionManager.getConfiguration().isInWhiteList(callPurposeForResetIntent)
then return the callPurposeForResetIntent

5. If callPurposeForResetIntent is null or not on the white list, then a new String is tried for steps 2 through 4:
String leafUtterance = getNMostRecentChunkGroups(String personID, 1)
The String leafUtterance will be handled using steps 2 through 4 above. If callPurposeForLeafUtterance is present in the white list, then return callPurposeForLeaf.

6. If callPurposeForLeafUtterance is either null or not on the white list, then    repeat step 5 using the String mostRecentPair which is obtained by calling the method getNMostRecentChunkGroups(String personID, 2). Again steps 2 through 4 are applied to this string.

========================================
