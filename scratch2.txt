Paper documents are scanned into images (maps) of varying pixel widths and heights:
scanned paper image width:  pw
scanned paper image height: ph
These images are then displayed on an editing device within a window that is 500 pixels wide by 260 pixels high. Upon this editing surface icons representing various devices are placed.
Let the coordinates of a device on such a surface be dx, and dy. The origin (0,0) is at the top left of the editing surface. x increases to the right and y increases downward. The requirement is for the 500X260 surface that contains the icons to be correctly placed on the screen of a hardware device:
width of screen = sw
height of screen = sh
The goal is to maximize the size of the image displayed on the hardware screen. Since the hardware screen UI is written in angularJS, CSS, and html, one way to do this is to apply this CSS style to the 500X260 background image:
 <div>
     style="
     position: absolute;
     top: 0;
     left: 0;
     height: 100%;
     width: 100%;
     background-size: contain;
     background-position: center center;
     background-repeat: no-repeat;
     ng-style="{'background-image': 'url(' + siteURL + 'php/getImage.php?sid=' + sid + '&id=' + map.media_id + ')'}">
</div>


The goal is to write a javascript function that takes as inputs pw, ph, dx, dy, sw, sh, and returns fx and fy. The challenge is to return the value of fx as a percentage of sw, and fy as a percentage of sh. In other words, the position of each icon should be given as a percentage of the width (sw) and height (sh) of the hardware screen.

To summarize: the paper image with size pw x ph is first fitted into an area of 500 x 260 and then the icons are placed on the 500 x 260 area. The the image on the 500 x 260 area is used as a "contain" style background on the final viewing device of sw x sh dimensions

Demonstrate that the returned percentage values will correctly map the icon from its original position of dx and dy on the original screen to the final position on the output screen. 

Question: Does the final position fx, fy depend on the original size of the paper map ?  or fx and fy only depend on the 500 x 260 area? Explain.


$rootScope.mapIconPosition = function (pw, ph, dx, dy, sw, sh) {
    referenceWidth = 500;
    referenceHeight = 260;
    // Step 1
    let spw = referenceWidth / pw;
    let sph = referenceHeight / ph;
    let smap = Math.min(spw, sph);

    // Step 2
    let dx_eff = dx + (referenceWidth - pw * smap) / 2;
    let dy_eff = dy + (referenceHeight - ph * smap) / 2;

    // Step 3
    let ssw = sw / referenceWidth;
    let ssh = sh / referenceHeight;
    let sfinal = Math.min(ssw, ssh);

    // Step 4
    let fx = dx_eff * sfinal;
    let fy = dy_eff * sfinal;

    // Step 5
    let fx_centered = fx + (sw - referenceWidth * sfinal) / 2;
    let fy_centered = fy + (sh - referenceHeight * sfinal) / 2;

    // Step 6
    let fxPercent = (fx_centered / sw) * 100;
    let fyPercent = (fy_centered / sh) * 100;
    
    return { fxPercent, fyPercent };
}

// Test the function
let result = mapIconPosition(1000, 520, 200, 130, 1920, 1080);
console.log("Icon should be placed at:", result.fxPercent + "% of screen width,", result.fyPercent + "% of screen height");
